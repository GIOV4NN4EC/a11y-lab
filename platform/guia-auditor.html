Guia do Auditor — CursoVivo

Esta plataforma fictícia contém problemas reais de acessibilidade espalhados pelo fluxo natural de uso. Use este guia para validar suas descobertas ou como gabarito em treinamentos.

Fluxo sugerido de teste

  1. Comece em platform/index.html (landing + login)
  2. Faça login com qualquer e-mail → vai para o dashboard
  3. Navegue: Dashboard → Cursos → Detalhe do curso → Relatórios → Configurações
  4. Teste com teclado, leitor de tela e DevTools Accessibility

Mapa de problemas por critério

1.1.1 — Conteúdo Não Textual

PáginaElementoProblemaCorreção esperada
Landing Logo no header Imagem sem alt alt="CursoVivo" ou alt vazio se decorativo com texto adjacente
Landing Ícones dos recursos Ícones decorativos com alt longo e redundante alt="" e aria-hidden="true"
Dashboard Botões do topbar (sino, ajuda) Ícones sem nome acessível aria-label descritivo em cada botão
Cursos Thumbnails LGPD e UX Writing Imagens informativas sem alt Alt descrevendo o curso representado
Detalhe Banner do curso alt="imagem do curso" genérico Alt específico: "Banner do curso Acessibilidade Web"
Relatórios Gráfico de barras Gráfico informativo sem alternativa Alt resumindo tendência + tabela de dados
Relatórios Botão exportar Ícone sem rótulo aria-label="Exportar relatório"

1.3.1 — Informações e Relações

PáginaElementoProblemaCorreção esperada
Dashboard Tabela de alunos Dados em divs, não em table/th Tabela semântica com th scope
Dashboard Atividade recente Lista em divs ul/li ou lista semântica equivalente
Detalhe Seções do curso Salto de h1 para h4 Hierarquia sequencial h2 → h3
Configurações Campo Departamento span visual sem label for label for="departamento" associado ao input
Configurações Checkboxes de notificação Grupo sem fieldset/legend fieldset com legend descritiva

2.4.6 — Cabeçalhos e Rótulos

PáginaElementoProblemaCorreção esperada
Landing Campo de e-mail no login Só placeholder, sem label label for "E-mail corporativo"
Dashboard Busca no topbar Campo sem rótulo label ou aria-label descritivo
Dashboard Painel de alunos Cabeçalho "Detalhes" vago "Alunos em progresso" ou similar
Detalhe Card lateral Título "Mais informações" genérico "Inscrição no curso" ou equivalente
Detalhe Campo cupom Label "Campo opcional" "Cupom de desconto (opcional)"
Detalhe Botão de inscrição Texto "OK" ambíguo "Confirmar inscrição"
Configurações Seção do formulário Cabeçalho "Dados" vago "Informações do perfil"
Configurações Campo cargo Label "Informação 1" "Cargo" ou "Função na empresa"

Checklist rápido

  • 1.1.1 Todas as imagens informativas têm alt descritivo?
  • 1.1.1 Ícones decorativos estão ocultos do leitor de tela?
  • 1.1.1 Botões só-ícone têm nome acessível?
  • 1.3.1 Tabelas usam table, th, td com cabeçalhos?
  • 1.3.1 Listas usam elementos de lista semânticos?
  • 1.3.1 Campos de formulário têm label programaticamente associado?
  • 2.4.6 Cabeçalhos de seção descrevem o conteúdo?
  • 2.4.6 Rótulos de formulário comunicam o propósito do campo?
